# Лабораторна робота №6: Етичний хакінг власного застосунку

## Опис

Ця папка містить навчальний проєкт на Python (GUI), який демонструє вразливість **SQL Injection** та методи захисту від неї.

* `main.py` - Головний файл (графічний інтерфейс).
* `db_manager.py` - Модуль бази даних (створення та наповнення).
* `vulnerabilities.py` - Модуль логіки (вразливі та захищені функції, WAF).

## Технічне завдання

Програма емулює роботу університетської бази даних студентів та реалізує наступний функціонал:
* **Вразливий режим:** використання конкатенації рядків (f-strings) для демонстрації витоку даних (`Data Leak`) та обходу авторизації (`Auth Bypass`).
* **Захищений режим:** використання параметризованих запитів (`Prepared Statements`) різних типів (`?` та `:name`).
* **Бонус (IDS):** система логування спроб атак з кольоровим маркуванням.
* **Бонус (WAF):** вбудований фільтр вводу, що блокує небезпечні сигнатури (`UNION`, `' OR`, `--`) до виконання запиту.

**Технологія:** Python 3, Tkinter (GUI), SQLite3.

## Запуск програми

Програма використовує лише стандартні бібліотеки Python.

1.  Переконайтесь, що у вас встановлено **Python 3.x**.
2.  Перейдіть до поточної папки (`lab06`) у терміналі.
3.  Запустіть головний скрипт командою:
    ```bash
    python main.py
    ```
4.  **Сценарії тестування:**
    * **Вкладка 1 (Пошук):** Введіть пейлоад `' OR '1'='1` у червоне поле для демонстрації витоку всієї бази.
    * **Вкладка 2 (Логін):** Введіть `admin' --` для входу без пароля.
    * **Вкладка 3 (Логи):** Перегляньте зафіксовані спроби атак.
