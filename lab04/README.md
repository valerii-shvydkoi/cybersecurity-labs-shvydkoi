# Лабораторна робота №4: Особистий цифровий підпис

## Опис

Ця папка містить проєкт із графічним інтерфейсом (GUI) на Python, який реалізує систему цифрового підпису (ЕЦП) на основі алгоритму RSA.

* `main.py` - Головний файл запуску програми (графічний інтерфейс на Tkinter).
* `crypto_engine.py` - "Двигун", що містить власну реалізацію математики RSA, генерацію ключів та логіку підпису.

## Технічне завдання

Програма реалізує повний цикл роботи з цифровими підписами. Вона виконує наступні завдання, визначені у методичних рекомендаціях:
* **Власна реалізація RSA:** програма використовує "чисту" математику (тест Міллера-Рабіна, розширений алгоритм Евкліда) для роботи з ключами, без використання готових крипто-бібліотек.
* **Персоналізація:** генерація ключів відбувається на основі персональних даних (ПІБ, дата народження, секретне слово), які використовуються як джерело ентропії (seed).
* **Підписання та верифікація:** реалізовано алгоритми створення підпису (шифрування хешу приватним ключем) та його перевірки.
* **Бонус (GUI):** розроблено зручний графічний інтерфейс користувача (на базі `tkinter`).
* **Бонус (Мульти-хешування):** підтримка різних алгоритмів хешування (`SHA-256`, `SHA-512`, `MD5`).

**Технологія:** Python 3, Tkinter (GUI), Hashlib (стандартна бібліотека).

## Запуск програми

Програма використовує лише стандартні бібліотеки Python, тому встановлення сторонніх пакетів не потрібне.

1.  Переконайтесь, що у вас встановлено **Python 3.x**.
2.  Перейдіть до поточної папки (`lab04`) у терміналі.
3.  Запустіть головний скрипт командою:
    ```bash
    python main.py
    ```
4.  Використовуйте графічний інтерфейс:
    * **Вкладка 1:** Введіть персональні дані та згенеруйте ключі.
    * **Вкладка 2:** Оберіть файл та створіть цифровий підпис.
    * **Вкладка 3:** Перевірте оригінальний або модифікований файл для підтвердження цілісності.
